## 0. Panel 判定快照

- **As-is verdict**：Borderline / Weak Reject。
- **可修复性**：无不可修复 CRITICAL；主要风险集中在方法透明度、probe 统计、选择偏差、跨模型外部效度、claim 边界和 AAAI 格式/复现材料。
- **目标修后姿态**：Weak Accept 级别；保留核心故事，但把证据链、防御和 venue 合规补齐。
- **最高优先级原则**：先补能改变审稿分数的证据与透明度，再做表达和格式。

## P0 — 不修大概率被拒

### P0-1 补全 probe / SFT 方法透明度

**Spec**

AAAI reviewer 需要能从正文 + 附录重建关键实验流程。当前稿件只声明结果可从 append-only result store 和 scripts 再生，但主文缺少足够的 construction detail。

**Steps**

- [x] 在 `Experimental Setup` 增加 SFT pipeline 摘要：teacher、train size、trace filtering、LoRA/QLoRA 或 dense control 配置、训练步数、decode 设置。
- [x] 增加 probe construction 算法框：CoT 抽取、on-path numeric intermediate/operand 选择、wrong value 分布、force-continue prompt、mask-image 设置。
- [x] 明确 numeric grading tolerance：单位、四舍五入、百分比、等价表达、解析失败如何处理。
- [x] 明确 `snap / follow / other` 自动判定规则，并列 3-5 个边界样例。
- [x] 附录或 supplementary 增加 artifact table：result JSON、script、seed、checkpoint、hash 或路径。

**Acceptance**

- [x] reviewer 不需要读代码也能理解每个 headline table/figure 如何生成。
- [x] 每个 probe metric 都能追溯到一个明确输入集合、过滤条件和判定函数。

### P0-2 给 faithfulness probe 补统计检验

**Spec**

Panel 共识认为 SFT accuracy 的 McNemar 统计较充分，但 faithfulness probe 主要是比例对比。AAAI 需要对 `corrupt - shuffle`、follow、snap、gain subset 给不确定性。

**Steps**

- [ ] 对每个 cell 报 `F = flip_corrupt - flip_shuffle` 的 95% CI。
- [ ] 对 follow rate、snap rate 报 binomial CI 或 bootstrap CI。
- [ ] 对 corrupt vs shuffle 做 paired bootstrap / exact randomization / McNemar-style paired test（按数据结构选择一种并说明）。
- [ ] 对多 cell headline probe 做 Holm 或 BH correction，或明确哪些是 confirmatory、哪些是 diagnostic。
- [ ] gain-subset 不只报 TabMWP；若 ChartQA/FinQA 不报，说明原因。
- [ ] 在 Table 1 或补充表中加入 `n_raw -> n_probe` 过滤流。

**Acceptance**

- [ ] “corrupt 不比 shuffle 更 flip”不再只是裸比例判断。
- [ ] “follow near zero”带上界，避免 under-powered null 被攻击。

### P0-3 处理 probe 样本选择偏差

**Spec**

当前 probe 主要在 “correctly answered with emitted CoT” 子集上运行。DA 和方法 reviewer 都认为这会造成 survivorship / selection bias。

**Steps**

- [ ] 报告每个任务的样本流：raw 400、可解析 CoT、base correct、SFT correct、probe eligible。
- [ ] 分层报告 stable-correct、SFT-gain、SFT-loss、both-wrong 的 probe 结果；至少给 headline task。
- [ ] 说明无 CoT、异常 CoT、解析失败样本如何处理。
- [ ] 对 selection 影响做 sensitivity analysis：例如把 failed parse 作为 non-follow / unknown 的上下界。
- [ ] 讨论只在正确样本上测 written-chain load-bearing 的合理性和边界。

**Acceptance**

- [ ] reviewer 不能用“你只测了最不依赖 CoT 的样本”一句击穿主结论。

### P0-4 加更干净的 control，缓解 shuffle 过强问题

**Spec**

Panel 认为 `shuffle` 可能破坏语义、顺序和 token 分布，不是完全干净的 format-matched control。需要增加更温和、更局部的 control。

**Steps**

- [ ] 增加无关数字替换 control：替换非 on-path 数字，保持局部语法。
- [ ] 增加 format-preserving local perturbation：单位、标点、同长度数字、局部 token 替换。
- [ ] 增加语义一致反事实重写：整条 CoT 内部一致但中间数错误，结论不直接改写。
- [ ] 区分三类条件：format disruption、semantic inconsistency、causal numeric intervention。
- [ ] 在正文解释为什么 shuffle 仍作为 harsh control 有价值，但不再作为唯一 control。

**Acceptance**

- [ ] 若低 follow 仍存在，结论从“模型不信任 OOD shuffle/corrupt”升级为“在更温和反事实下 written intermediate 仍不承载答案”。

### P0-5 补跨模型 / 跨家族外部效度

**Spec**

当前主要是 Qwen3-VL 8B/32B；领域/EIC reviewer 都认为外部效度不足。至少需要一个非 Qwen 家族关键复现，或显著收窄 claim。

**Steps**

- [ ] 优先选择一个可本地跑的非 Qwen VLM（如 InternVL/LLaVA/Qwen2.5-VL 以外家族，按实际可用性定）。
- [ ] 在一个 headline substrate 上复现最小 probe：accuracy + corrupt/shuffle + snap/follow。
- [ ] 若算力/模型不可行，在 Limitations 中明确“single model family”并把标题/摘要 claim 收窄。
- [ ] 表中区分 Qwen-family evidence 和 cross-family evidence。

**Acceptance**

- [ ] AAAI reviewer 不能轻易把结果归因为 Qwen3-VL 或 QLoRA 特定行为。

### P0-6 收窄 “Reading, Not Reasoning” 的 claim 边界

**Spec**

标题可保留，但正文必须持续说明因果结论只针对 emitted written rationale，不等于证明没有 latent computation 或没有 reasoning。

**Steps**

- [ ] Abstract 第一段加入限定：causal claim concerns emitted written chains, not hidden-state computation。
- [ ] Setup 给操作性定义：`reading`、`load-bearing written chain`、`latent reasoning` 的边界。
- [ ] Discussion 明确：视觉重读本身可能包含 latent arithmetic；本文只证明答案不从被篡改的 written intermediate 读出。
- [ ] FinQA 段把 “not recomputation” 改为 “not recomputation from emitted operands under this probe”。
- [ ] 全文查找过强措辞：`not reasoning`、`internalization failed`、`does not compute`，必要处降调。

**Acceptance**

- [ ] 最强反论点只能攻击 scope，不再能说作者混淆 written rationale 和 hidden computation。

## P1 — 强化说服力与 AAAI 定位

### P1-1 增加 field positioning table

**Spec**

领域 reviewer 认为相关工作偏压缩。需要一张定位表，把本文与 CoT faithfulness、visual rationale、tool/program distillation、latent reasoning、chart/table QA 区分开。

**Steps**

- [ ] 增加一张 related-work positioning table。
- [ ] 行维度建议：text CoT / visual evidence / tool trace / latent state / chart-table QA。
- [ ] 列维度建议：object audited、intervention type、model setting、what conclusion is allowed。
- [ ] 补 CoT 基础文献簇：Wei CoT、Kojima zero-shot CoT、self-consistency。
- [ ] 补 rationale faithfulness / explanation 文献簇：Jacovi & Goldberg、Wiegreffe & Marasovic、ERASER 等。
- [ ] 补 chart/table QA 和 tool/program distillation 文献簇：DVQA/FigureQA/PlotQA/TAT-QA/FinQA、PAL/Program-of-Thoughts/Toolformer/distilling step-by-step。

**Acceptance**

- [ ] AAAI 广义 AI reader 能一眼看出本文不是重复 Lanham-style corrupt probe，而是在 VLM distillation + fixed visual evidence 场景中做 causal audit。

### P1-2 证明或降调 “re-read visible evidence” 机制解释

**Spec**

DA 指出 snap 不一定证明 final step 视觉重读，也可能来自语言先验、答案残留或 consistency behavior。

**Steps**

- [ ] 增加局部遮挡或 counterfactual chart/table editing：只改相关视觉单元格/柱子/表格 cell。
- [ ] 或在正文把 “re-read” 改成 “answer remains recoverable from visible evidence and the model snaps to it”。
- [ ] 对 masked condition 做更清楚解释：masked 后 other 上升但 follow 仍小，说明不是简单服从坏 chain。
- [ ] 若有 attention/activation evidence，不作为主证据，只作为辅助。

**Acceptance**

- [ ] 机制语言不超过证据；若保留 “re-read”，有直接视觉反事实支持。

### P1-3 补部署审计和误用讨论

**Spec**

Perspective reviewer 给 Weak Accept 的原因是实践影响强，但指出治理/部署讨论薄。AAAI 版本应说明该 probe 如何服务评测，而不是只做批判。

**Steps**

- [ ] Discussion 加 “Deployment Audit Implications” 小节。
- [ ] 给最小 checklist：when to show CoT、when to suppress/label CoT、faithfulness metric threshold、risk disclosure。
- [ ] 增加误用风险：不能用“不忠实 CoT”作为不提供解释的借口，也不能继续展示不忠实 CoT 制造透明感。
- [ ] 说明目标用户：benchmark authors、VLM system builders、XAI auditors、high-stakes table/chart QA deployers。

**Acceptance**

- [ ] 论文从“批判 CoT SFT claims”升级为“给出 causal audit practice”。

### P1-4 明确 FinQA / TallyQA 与主线的关系

**Spec**

DA 指出 TallyQA 是 base model probe，不完全同构于 CoT SFT gains；FinQA conclusion-copying 不证明训练增益不是 latent program learning。

**Steps**

- [ ] TallyQA 标为 external regime check，而非与 ChartQA/TabMWP 同等 SFT 机制证据。
- [ ] FinQA 拆成两个结论：operand-follow near zero；conclusion-line follow high。
- [ ] 避免说 FinQA 证明 “not recomputation” 的广义命题。
- [ ] 在 figure/table label 中标注 each regime tests what source: pixels / redundant text / conclusion line。

**Acceptance**

- [ ] reviewer 能接受各 substrate 是互补诊断，不会指责作者把异质实验硬堆成一个因果证明。

### P1-5 将 `Experimental Setup` 改为方法贡献导向标题

**Spec**

当前 Section 3 叫 `Experimental Setup`，容易让 reviewer 以为这里只是 datasets / models / metrics 的配置说明。但该节实际定义了本文的核心审计方法：干预对象、`corrupt / shuffle`、`snap / follow / other` readout、load-bearing 判据和统计口径。应把标题改成方法贡献导向。

**Steps**

- [ ] 将 `\section{Experimental Setup}` 改为 `\section{Method: Causal Faithfulness Audit}` 或更短的 `\section{Causal Faithfulness Audit}`。
- [ ] 保留原 label（如 `\label{sec:setup}`）或同步更新引用，避免交叉引用断裂。
- [ ] 调整小段落顺序为：Tasks / Models / Probe design / Readout metrics / Statistics / Robustness controls。
- [ ] 在节首加 1-2 句总述：本节不仅列实验设置，还定义 causal audit 如何判定 emitted written chain 是否 load-bearing。
- [ ] 若篇幅允许，把 `corrupt / shuffle / snap / follow` 定义提前到该节开头，而不是在表注或结果中首次解释。

**Acceptance**

- [ ] Section 3 的标题和内容明确传达“审计方法”是本文贡献之一，而不是普通实验设置。
- [ ] AAAI reviewer 能从目录结构看出本文提出/实例化了一套 causal faithfulness audit。

## P2 — 格式、匿名和复现合规

### P2-1 切 AAAI 模板并复查页限

**Spec**

当前 `paper.tex` 是 11pt article 单栏，不是 AAAI 双栏模板。AAAI main track 按官方要求控制技术正文页数，参考文献页另计。

**Steps**

- [ ] 切换到 AAAI 官方 LaTeX 模板。
- [ ] 复查技术正文页数；目标不超过 7 页技术内容。
- [ ] 参考文献可另起页，但只放 references。
- [ ] 重新检查 Figure/Table 宽度、caption 字号、float 位置。
- [ ] 确认 supplementary 中的信息不被主文必要论证依赖。

**Acceptance**

- [ ] `paper.pdf` 在 AAAI 模板下页限合规，且 headline figures/tables 不溢出。

### P2-2 双盲和元数据检查

**Spec**

AAAI 双盲投稿需要移除作者身份线索。当前正文有 Anonymous Authors，但仍需检查 footnote、仓库路径、PDF metadata、artifact links。

**Steps**

- [ ] 删除或匿名化 author footnote 中可能识别身份的信息。
- [ ] 检查 `hyperref` metadata、PDF author/title、file path 泄露。
- [ ] 代码/数据链接使用匿名仓库或提交系统允许的匿名 supplement。
- [ ] 避免 “our released scripts” 指向非匿名位置；可写 “included in the anonymous supplementary material”。

**Acceptance**

- [ ] PDF 和 supplementary 不包含可识别作者、机构、用户名、私有路径或非匿名仓库。

### P2-3 Reproducibility checklist 和 artifact appendix

**Spec**

AAAI 需要 reproducibility checklist。Panel 认为当前 reproducibility statement 过短。

**Steps**

- [ ] 填 AAAI reproducibility checklist。
- [ ] 附录列出 dataset splits、license/availability、model checkpoints、hardware、training time、random seeds。
- [ ] 列出 result store schema 和 append-only policy。
- [ ] 给最小复现实验命令：build data、eval SFT、run probe、regen tables/figures。
- [ ] 标明哪些 artifact 随匿名 supplement 提交，哪些因许可需用户自行下载。

**Acceptance**

- [ ] 复现审稿人能从 checklist + appendix 判断结果可重建性，而不是只看到一句声明。

## P3 — 可选但高收益

### P3-1 增加生成时序更接近的 intervention

**Spec**

DA 最强反论点是 force-continue after corrupt 不是原始 forward path 的因果干预。若时间允许，增加更接近生成时序的 probe。

**Steps**

- [ ] 设计 online replacement：模型生成到目标中间数附近时替换 token 后继续。
- [ ] 设计 prefix-only condition：只给 rationale 前缀，不给结论行。
- [ ] 设计 no-visual / locked-visual condition：区分回读证据与使用文本前缀。
- [ ] 与现有 force-continue 结果并列报告，不替换主结果。

**Acceptance**

- [ ] 即使 reviewer 不完全接受 force-continue，也有一个更接近 causal path 的 robustness probe。

### P3-2 人类信任 / 用户披露微型分析

**Spec**

Perspective reviewer 建议补 human-centered angle。若篇幅允许，可用小型情景分析替代真实用户实验。

**Steps**

- [ ] 增加 1 段 explanation-induced overtrust 风险。
- [ ] 给 2-3 个应用场景：financial table QA、education chart analysis、scientific figure QA。
- [ ] 说明何时应该显示 structured evidence 而不是 free-form CoT。

**Acceptance**

- [ ] Broader impact 不只是泛泛而谈，与本文实验证据直接相连。
