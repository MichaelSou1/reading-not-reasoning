## P3 — 可选但高收益

### P3-1 增加生成时序更接近的 intervention

**Spec**

DA 最强反论点是 force-continue after corrupt 不是原始 forward path 的因果干预。若时间允许，增加更接近生成时序的 probe。

**Steps**

- [ ] 设计 online replacement：模型生成到目标中间数附近时替换 token 后继续。
- [ ] 设计 prefix-only condition：只给 rationale 前缀，不给结论行。
- [ ] 设计 no-visual / locked-visual condition：区分回读证据与使用文本前缀。
- [ ] 与现有 force-continue 结果并列报告，不替换主结果。

**Acceptance**

- [ ] 即使 reviewer 不完全接受 force-continue，也有一个更接近 causal path 的 robustness probe。

### P3-2 人类信任 / 用户披露微型分析

**Spec**

Perspective reviewer 建议补 human-centered angle。若篇幅允许，可用小型情景分析替代真实用户实验。

**Steps**

- [ ] 增加 1 段 explanation-induced overtrust 风险。
- [ ] 给 2-3 个应用场景：financial table QA、education chart analysis、scientific figure QA。
- [ ] 说明何时应该显示 structured evidence 而不是 free-form CoT。

**Acceptance**

- [ ] Broader impact 不只是泛泛而谈，与本文实验证据直接相连。
